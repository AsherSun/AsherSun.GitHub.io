# HTML4常用标签及其含义

<!DOCTYPE >是网页声明，用来告知 Web 浏览器页面使用了哪种 HTML 版本

<!DOCTYPE html> 是html5标准网页声明

<html>&</html>限定文档的开始点和结束点

<head>&</head>网页制作的文件头标记，用于标识网页，方便搜索

<mate charset="utf-8">表示字符编码格式是utf-8，其它字符编码格式还有gb2312（国家标准第2312条，不含繁体字）、gbk（国家标准扩展版，含繁体字）、unicode（万国码）等，uft-8是unicode的升级版，比较常用。

<title>&</title>描述文档的标题，通常放在<head>&</head>标签中

<body>&<body>描述的是页面内容（可见的）<head></head>标签是同级

<h1> ~ <h6>.....&</h1>,</h2>.....一级标题，二级标题.....

<p>&</p>段落

<a>&</a>超链接，用户链接网络的各个页面

<!-- 注释 --> 注视标签, 用户在HTML文档中注视内容，此功能可以方便后续开发人员理解该处业务逻辑

<hr/>水平线

<br>&<br />折行

<b>&</b> <strong>&</strong>文本加粗、<strong> 告诉浏览器这个标签里面的内容很重要

<em>&</em> <i>&</i>字体倾斜

<img src="资源地址" alt="图片描述信息" width=".." height=".." /> 用户图片显示

<table>&</table>一个表格的开始和结束

<tr>&</tr>表格中的一行开始和结束

<td>&</td>行中的数据开始和结束

<th>&</th>表头

<caption>&</caption>表格的标题

<ul>&</ul> <li>&</li>无序列表

<ol>&</ol> <li>&</li>有序列表（默认123...，加type="..."定义成别的类型）

自定义列表以 <dl> 标签开始。每个自定义列表项以<dt>开始。每个自定义列表项的定义以<dd> 开始

# img 标签详解

img 标签用户展示网页中的图片，让网页的展现形式多了一种方式

# img 标签中的各种属性

  • width 用于控制图片的宽度
  • height 用于控制图片的高度
  • alt 对该图片的信息描述，常用于网页SEO优化。
  • src 用于显示图片，通常我们给图片的地址就可以了

例子：

<img src="https://dss1.bdstatic.com/kvoZeXSm1A5BphGlnYG/skin/256.jpg?2" alt='我是一张图片' width='300' height='300'/>

# 相对路径与绝对路径

img标签中的src属性需要的值为图片的资源地址。那么该地址的具体写法应该规避哪些问题呢？

相对路径含义

指由这个文件所在的路径引起的跟其它文件（或文件夹）的路径关系。使用相对路径可以为我们带来非常多的便利

绝对路径含义

绝对路径是指目录下的绝对位置，直接到达目标位置，通常是从盘符开始的路径。

例子：

<!-- 绝对路径 -->
<img src="https://dss1.bdstatic.com/kvoZeXSm1A5BphGlnYG/skin/256.jpg?2" />
<!-- 某一个磁盘下的绝对路径 -->
<img src="D:///xxx/Desktop/Asher/web_teching/01.jpg" />

<!-- 相对路径 -->
<img src="./xx/sdf.jpg"/>
<img src="xx/sdf.jpg" />
<!-- ./ 当前目录 -->
<!-- ../ 当前目录的上一级目录 -->

思考：在实际开发中为什么通常都使用相对路径？

  1. 在项目中使用绝对路径，当该项目移动到另一个磁盘中，会造成静态资源访问不到的情况
  2. 在项目中使用相对路径，则不会有这种问题
